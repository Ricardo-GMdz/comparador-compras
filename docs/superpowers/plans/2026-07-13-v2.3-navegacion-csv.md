# v2.3 — Navegación con sidebar + CSV resumido — Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Separar el buscador (Inicio, con favoritos) del directorio completo (Historial) con un sidebar, y reducir el CSV a 12 columnas legibles en español.

**Architecture:** El CSV es un cambio chico en `src/server/api.ts` (con test). La navegación es frontend puro en `web/`: un sidebar que muestra/oculta dos secciones (`#vista-inicio` / `#vista-historial`) sin router; el estado en memoria (`directorio`, `mejorOpcion`) se comparte y `render()` repinta. Spec: `docs/superpowers/specs/2026-07-13-v2.3-navegacion-csv-design.md`.

**Tech Stack:** el existente (TS/Node ESM, Hono, zod, vitest, vanilla JS). Sin dependencias nuevas.

**Convenciones:** TDD donde hay lógica (test primero → FALLAR → implementar → PASAR → commit). Comandos con prefijo `rtk`. Comentarios en español, identificadores en inglés. Inmutabilidad. Antes de cada commit: `rtk pnpm run typecheck && rtk pnpm run lint && rtk pnpm test` en verde. Rama: `feat/v2.3-navegacion-csv` (ya creada).

function toCsv(suppliers: readonly Supplier[]): string {
  const rows = suppliers.map((s) =>
    [
      s.name,
      s.website,
      s.material,
      s.region,
      s.wholesalePrice ?? s.catalogPrice,
      s.currency,
      s.contact.email,
      s.contact.whatsapp,
      s.contact.phone,
      s.address,
      s.status,
      s.favorite ? "sí" : "",
    ]
      .map(csvField)
      .join(","),
  );
  return [CSV_COLUMNS.join(","), ...rows].join("\n");
}
