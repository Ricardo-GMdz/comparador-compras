# Plan de Implementación — v2.1: gestión, cotización, calidad de dato, higiene

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convertir la app de sourcing (v2) en herramienta de trabajo: gestión del directorio (estado/notas/borrar/filtrar), pedido de cotización copiable, unidad de precio en el ranking, enriquecimiento manual de contactos, y cierre de deudas de higiene/robustez.

**Architecture:** Extiende los módulos existentes sin romper contratos: `domain/supplier.ts` gana `status` y `priceUnit`; `directory/store.ts` gana `updateSupplier`/`removeSupplier`; `ranking` compara por unidad dominante y la API filtra descartados; `server/api.ts` gana `PATCH/DELETE /api/proveedor/:key`, `POST .../enriquecer` y `GET /api/directorio.csv`; `web/` gana toolbar de gestión y modal de cotización. Spec: `docs/superpowers/specs/2026-07-04-v2.1-mejoras-design.md`.

**Tech Stack:** el existente (TS/Node ESM, Hono, zod, vitest, vanilla JS). Sin dependencias nuevas.

**Convenciones para TODAS las tareas:** TDD (test primero → correr y ver FALLAR → implementar → correr y ver PASAR → commit). Comandos con prefijo `rtk`. Comentarios en español. Inmutabilidad. Después de cada tarea: `rtk pnpm run typecheck && rtk pnpm run lint && rtk pnpm test` en verde y commit con mensaje convencional.

```ts
export function buildQuoteMessage(input: {
  supplierName: string;
  material: string;
  quantity: string;
```

```ts
  return [

    "",
    `Estoy buscando: ${input.spec} — cantidad: ${input.quantity}.`,

    "",
    "Quedo atento. ¡Gracias!",

}
```
